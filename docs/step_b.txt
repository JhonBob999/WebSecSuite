Этап B — Финальные поля + работа с редиректами

1) Сохранение результатов в объект задачи
   - В ScraperRunnable (core/scraper/runnables.py) после выполнения запроса:
     • результат запроса присваивается self.task.result = result.
     • это позволит всегда иметь последние данные внутри TaskManager.

2) Расширение краткого отображения результатов
   - Метод _format_result_short(payload):
     • добавляем отображение final_url, размера контента (content_len),
       времени запроса (timings.request_ms).
     • считаем количество редиректов в redirect_chain и выводим
       как r=N.
   - Пример строки: "200 · 12KB · 120ms · r=2".

3) Обновление ячейки Results
   - set_results_cell(row, payload):
     • текстовое резюме (код + размер + время + редиректы).
     • tooltip = prettified JSON (весь payload).
   - при двойном клике всё ещё открываем JSON viewer.

4) Интеграция redirect_chain
   - В ScraperRunnable включаем follow_redirects=True.
   - В payload/result добавляем redirect_chain (список всех hop’ов).
   - В логах:
     • отдельный блок с цепочкой редиректов.
     • tooltip в колонке Status показывает финальный URL и hops.

5) Экспорт данных
   - В core/scraper/exporter.py:
     • в CSV/JSON/XLSX добавляем новые поля:
       final_url, status_code, content_len, timings.request_ms, redirect_chain.
     • исправляем потерю колонок, если первая задача без этих полей.
   - JSON экспортирует полный словарь без потерь.
   - Проверка: сравнить данные в Result tooltip и в экспортированном файле.

6) UI-обновления
   - В логах при [RESULT] теперь показываем финальный URL,
     время запроса и редиректы.
   - tooltip в колонке Status дополняем инфой о редиректах.
   - автоподгонка ширины колонок (Results, Status).

7) Тестирование
   - Запустить задачи на URL с редиректами (например, http → https).
   - Проверить:
     • в логах видна цепочка переходов;
     • в Results корректно подсчитан r=N;
     • экспорт содержит redirect_chain как массив.
   - Проверить задачи без редиректов — корректно отображается r=0.

Итог:
   - Все задачи теперь содержат расширенные данные.
   - В колонке Results — краткое резюме.
   - В tooltip — полный JSON с финальными полями.
   - Экспорт без потерь, поддерживает redirect_chain.
